🎮 Lesson 1: Unity Basics

Master the fundamentals of Unity in 1 hour

⏱️ Total Time: 60 Minutes

This lesson is designed to give you hands-on Unity experience in just one hour. Follow along step by step!

🔹 0–10 min: What is Unity?

✅ Goal:

Understand what Unity is and why we use it

🧠 Key Concepts:

  • Unity = Game Engine (used to build games, apps, simulations)
  • Supports:
    • 2D & 3D games
    • Mobile, PC, Console
  • Uses C# as the scripting language

💡 Simple Understanding:

Unity = Editor + Engine + Physics + Rendering + Scripting

Why Unity? Unity is one of the most popular game engines used by indie developers and AAA studios alike. It's beginner-friendly yet powerful enough to create commercial games!

🔹 10–25 min: Unity Interface (VERY IMPORTANT)

✅ Goal:

Get comfortable with Unity windows (you'll use these daily)

🧩 Core Windows:

Window Purpose What You Do Here
Scene View Design workspace Where you design your game world
Game View Player perspective What the player actually sees
Hierarchy Object list List of all objects in the scene
Inspector Properties panel Modify selected object properties
Project Window Asset browser All assets (scripts, images, models)
Console Debug output Errors, logs, debug messages

💡 Analogy:

  • Hierarchy = Tree structure (parent-child relationships)
  • Inspector = Settings panel (like right-click properties)
  • Scene = Workspace (your design canvas)
  • Game = Final output (what players experience)
💡 Pro Tip: You can customize the Unity layout! Go to Window → Layouts to choose different arrangements. Start with the "Default" layout as a beginner.

🔹 25–40 min: GameObjects & Components

✅ Goal:

Understand Unity's core concept - the foundation of everything in Unity

🧠 The Most Important Concept in Unity:

👉 Everything in Unity = GameObject + Components

🧱 GameObject:

A GameObject is like an empty container (think of it as a box)

By itself, it does nothing. It's just a container waiting for components.

🔧 Components:

Components add functionality to GameObjects:

Component What It Does
Transform Position, rotation, scale (every GameObject has this!)
Sprite Renderer Makes object visible (2D graphics)
Rigidbody Adds physics (gravity, forces, movement)
Collider Enables collision detection
Script Custom behavior you program

💡 Real Example:

Example: A Player Character
GameObject: Player

Components attached:
  • Transform (position: x=0, y=0, z=0)
  • Sprite Renderer (displays player image)
  • Rigidbody2D (adds physics)
  • BoxCollider2D (collision detection)
  • PlayerController (custom script for movement)
💡 Think of it this way: GameObject = Car body, Components = Engine, wheels, steering wheel, seats. Without components, the car (GameObject) is just an empty shell!

🔹 40–50 min: Hands-On - Building Your First Scene

🎯 Project: Circle Player + Square Platform

Learn components by building a simple physics-based scene!

📦 Step 1: Scene Setup

  1. Create a Circle (Player):
    • Hierarchy → Right-click → 2D Object → Sprites → Circle
    • Rename it to "Player" in the Inspector
  2. Create a Square (Platform):
    • Hierarchy → Right-click → 2D Object → Sprites → Square
    • Rename it to "Platform"
    • In Transform: Set Scale X = 5, Y = 0.5 (makes it look like a platform)
    • Position: X = 0, Y = -3 (move it down)

🧩 Step 2: What is a Component?

Component = A behavior or property attached to a GameObject

Important Rules:

  • Every GameObject MUST have a Transform component
  • All other components are optional
  • Components define what a GameObject can do

🎨 Step 3: Sprite Renderer Component

Controls how the GameObject looks visually:

  1. Select Platform in Hierarchy
  2. Look at Inspector → Find Sprite Renderer component
  3. Click Color → Change to dark blue
  4. See the Sprite field → shows "Square" (the shape)
Understanding Sprite Renderer Properties
Sprite: The actual image/shape to display
Color: Tint applied to the sprite
Order in Layer: Controls which objects appear in front
  • Higher number = appears in front
  • Lower number = appears behind
💡 Try This: Set Player's "Order in Layer" to 1, Platform to 0. This ensures the player always appears in front!

⚙️ Step 4: Add Physics (Rigidbody2D)

Make objects behave like real-world physics!

  1. Select Player in Hierarchy
  2. Inspector → Click "Add Component"
  3. Type "Rigidbody 2D" → Select it
  4. Press Play ▶️ → Watch the player fall!

🧠 What Just Happened?

Rigidbody2D makes the GameObject react to gravity and forces, just like real objects!

🔧 Adjust Gravity:

  1. Stop the game (click Play button again)
  2. Select Player → Inspector → Rigidbody2D component
  3. Find Gravity Scale → Change from 1 to 5
  4. Press Play → Player falls faster!

❌ The Problem:

⚠️ Problem: The player falls through the platform!

Why? Unity doesn't know the shape of objects yet. It can't detect collisions without colliders.

🛡️ Step 5: Add Colliders

Colliders define the physical shape for collision detection:

Add Circle Collider to Player:

  1. Stop the game
  2. Select Player
  3. Add Component → "Circle Collider 2D"

Add Box Collider to Platform:

  1. Select Platform
  2. Add Component → "Box Collider 2D"

Test It:

Press Play ▶️ → Player now lands on the platform! 🎉

🧩 Collider Components Explained:

Collider Type Best For
Circle Collider 2D Round objects (balls, coins, player)
Box Collider 2D Square/rectangular objects (platforms, walls)
Polygon Collider 2D Complex irregular shapes

🔍 Step 6: Adjust Collider Size

  1. Select Player → Inspector → Circle Collider 2D
  2. Change Radius from 0.5 to 0.3
  3. Press Play → Notice the collision area is smaller than the visual!
💡 Important: Collider size doesn't have to match the sprite size. You can make hit boxes smaller or larger for gameplay purposes!

🎾 Optional: Add Bounciness!

  1. Project window → Right-click → Create → 2D → Physics Material 2D
  2. Name it "Bouncy"
  3. Select it → Inspector → Set Bounciness to 0.7
  4. Select Player → Circle Collider 2D → Drag "Bouncy" material to Material field
  5. Press Play → Player bounces! 🎾

🔹 50–60 min: Custom Components (C# Scripts)

✅ Goal:

Create custom behavior using C# scripts - your own components!

💡 Important Concept:

Scripts ARE Components! When you write a C# script that inherits from MonoBehaviour, you're creating a custom component that can be attached to GameObjects.

🪜 Create Player Movement Script:

  1. Project window → Right-click → Create → C# Script
  2. Name it PlayerController (no spaces!)
  3. Double-click to open in Visual Studio Code

✍️ Add Movement Code:

PlayerController.cs - Horizontal Movement
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    public Rigidbody2D rb;
    public float moveSpeed = 5f;

    void Update()
    {
        // Get input (A/D or Left/Right arrows)
        float moveX = Input.GetAxis("Horizontal");

        // Move player left/right
        rb.velocity = new Vector2(moveX * moveSpeed, rb.velocity.y);
    }
}

🎯 Attach and Configure:

  1. Stop the game (if playing)
  2. Select Player in Hierarchy
  3. Drag PlayerController script onto Player (or Add Component)
  4. In Inspector, find the PlayerController component
  5. Drag the Player GameObject into the Rb field (this links the Rigidbody2D)
  6. Press Play → Use A/D or Arrow Keys to move!

🧠 Code Breakdown:

  • public Rigidbody2D rb; → Reference to the Rigidbody component
  • Input.GetAxis("Horizontal") → Gets keyboard input (-1 to 1)
  • rb.velocity → Controls the movement speed of the object
  • moveSpeed → You can adjust this in the Inspector!
⚠️ If input doesn't work:
File → Project Settings → Player → Other Settings → Active Input Handling → Change to "Both"
(This enables the old input system)

🎮 Extra: Platform Physics Challenge

Make the game more interesting:

  1. Select Platform
  2. Add Component → Rigidbody2D
  3. In Rigidbody2D component:
    • Constraints → Check Freeze Position X, Y
    • Check Freeze Rotation Z
    • Set Mass to 10
  4. Now when the player lands, the platform has weight but doesn't fall!

🎯 Gameplay Idea:

You could create a balancing game where:

  • Platform can rotate (unfreeze rotation)
  • Player must balance while collecting items
  • Add more platforms with different physics properties
