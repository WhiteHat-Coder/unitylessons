1. Comparison Operators 🔍

Concept:

Comparison operators compare two values and return true or false.

Operator Meaning Example Result
== Equal to 5 == 5 true
!= Not equal to 5 != 3 true
> Greater than 5 > 3 true
< Less than 5 < 3 false
>= Greater than or equal 5 >= 5 true
<= Less than or equal 3 <= 5 true
Example: Using Comparison Operators
int playerHealth = 50;
int maxHealth = 100;

Debug.Log(playerHealth == maxHealth); // false
Debug.Log(playerHealth < maxHealth); // true
Debug.Log(playerHealth >= 25); // true
Output:
False
True
True
⚠️ Common Mistake: Don't confuse = (assignment) with == (comparison)!
x = 5 → assigns 5 to x
x == 5 → checks if x equals 5

2. If-Else Statements 🚦

Concept:

If-else statements execute different code based on conditions.

Syntax:

if (condition)
{
    // code runs if condition is true
}
else
{
    // code runs if condition is false
}

📝 Simple If Statement:

Example: Check Player Health
int playerHealth = 30;

if (playerHealth < 50)
{
    Debug.Log("Warning: Low health!");
}
Output:
Warning: Low health!

📝 If-Else Statement:

Example: Check Game Status
int score = 85;

if (score >= 50)
{
    Debug.Log("You passed! 🎉");
}
else
{
    Debug.Log("Try again!");
}
Output:
You passed! 🎉

📝 If-Else If-Else (Multiple Conditions):

Example: Grade System
int score = 75;

if (score >= 90)
{
    Debug.Log("Grade: A");
}
else if (score >= 80)
{
    Debug.Log("Grade: B");
}
else if (score >= 70)
{
    Debug.Log("Grade: C");
}
else if (score >= 60)
{
    Debug.Log("Grade: D");
}
else
{
    Debug.Log("Grade: F");
}
Output:
Grade: C

💡 How It Works:

  1. Program checks first condition (score >= 90) → false, skip
  2. Checks second condition (score >= 80) → false, skip
  3. Checks third condition (score >= 70) → true, executes this block
  4. Skips all remaining conditions

3. Logical Operators 🧠

Concept:

Logical operators combine multiple conditions.

Operator Name Meaning Example
&& AND Both must be true a > 5 && b < 10
|| OR At least one must be true a == 5 || b == 10
! NOT Reverses true/false !(a > 5)

📝 AND Operator (&&):

Example: Check Multiple Conditions
int health = 80;
int ammo = 15;

if (health > 50 && ammo > 10)
{
    Debug.Log("Ready for combat! ⚔️");
}
else
{
    Debug.Log("Not ready!");
}
Output:
Ready for combat! ⚔️

💡 AND Truth Table:

Both conditions must be true:

  • true && true = true
  • true && false = false
  • false && true = false
  • false && false = false

📝 OR Operator (||):

Example: Multiple Win Conditions
int score = 100;
int timeBonus = 0;

if (score >= 100 || timeBonus > 0)
{
    Debug.Log("Victory! 🏆");
}
else
{
    Debug.Log("Keep playing!");
}
Output:
Victory! 🏆

💡 OR Truth Table:

At least one condition must be true:

  • true || true = true
  • true || false = true
  • false || true = true
  • false || false = false

📝 NOT Operator (!):

Example: Reverse Condition
bool isGameOver = false;

if (!isGameOver)
{
    Debug.Log("Game is still running");
}
else
{
    Debug.Log("Game over");
}
Output:
Game is still running

📝 Combining Logical Operators:

Example: Complex Conditions
int age = 25;
bool hasLicense = true;
int drivingExperience = 3;

if ((age >= 18 && hasLicense) && drivingExperience > 2)
{
    Debug.Log("Can rent a sports car! 🏎️");
}
else
{
    Debug.Log("Not eligible");
}
Output:
Can rent a sports car! 🏎️

4. Nested If Statements 📦

Concept:

Nested if statements = if statements inside other if statements

Used when you need to check multiple levels of conditions

Example: Game Login System
string username = "player123";
string password = "pass456";
bool hasInternet = true;

if (hasInternet)
{
    Debug.Log("Connected to server");
    
    if (username == "player123")
    {
        if (password == "pass456")
        {
            Debug.Log("✅ Login successful!");
        }
        else
        {
            Debug.Log("❌ Wrong password");
        }
    }
    else
    {
        Debug.Log("❌ Username not found");
    }
}
else
{
    Debug.Log("❌ No internet connection");
}
Output:
Connected to server
✅ Login successful!
💡 Better Alternative: You can rewrite nested ifs using logical operators:
if (hasInternet && username == "player123" && password == "pass456")
{
    Debug.Log("✅ Login successful!");
}

5. Switch Statements 🔀

Concept:

Switch statement = cleaner way to handle multiple specific values

Use when you have many if-else if statements checking the same variable

📝 Basic Switch Statement:

Example: Day of the Week
int day = 3;

switch (day)
{
    case 1:
        Debug.Log("Monday");
        break;
    case 2:
        Debug.Log("Tuesday");
        break;
    case 3:
        Debug.Log("Wednesday");
        break;
    case 4:
        Debug.Log("Thursday");
        break;
    case 5:
        Debug.Log("Friday");
        break;
    default:
        Debug.Log("Weekend!");
        break;
}
Output:
Wednesday

🧠 Switch Statement Structure:

  • switch (variable) - variable to check
  • case value: - specific value to match
  • break; - exit the switch (REQUIRED!)
  • default: - runs if no cases match (optional)

📝 Switch with Strings:

Example: Weapon Selection
string weapon = "sword";

switch (weapon)
{
    case "sword":
        Debug.Log("⚔️ Melee damage: 50");
        break;
    case "bow":
        Debug.Log("🏹 Ranged damage: 40");
        break;
    case "staff":
        Debug.Log("🪄 Magic damage: 60");
        break;
    default:
        Debug.Log("✊ Fist damage: 10");
        break;
}
Output:
⚔️ Melee damage: 50

📝 Multiple Cases, Same Action:

Example: Enemy Type
string enemy = "zombie";

switch (enemy)
{
    case "zombie":
    case "skeleton":
    case "ghost":
        Debug.Log("Weak enemy - 10 XP");
        break;
    case "dragon":
    case "demon":
        Debug.Log("Boss enemy - 100 XP");
        break;
    default:
        Debug.Log("Unknown enemy");
        break;
}
Output:
Weak enemy - 10 XP
⚠️ Important: Always include break; after each case! Without it, the code "falls through" to the next case (usually not what you want).

6. Ternary Operator (Shortcut) ⚡

Concept:

Ternary operator = shorthand for simple if-else statements

Syntax: condition ? valueIfTrue : valueIfFalse

Example: Compare If-Else vs Ternary
// Traditional if-else
int age = 20;
string status;

if (age >= 18)
{
    status = "Adult";
}
else
{
    status = "Minor";
}

// Ternary operator (same thing, shorter!)
string status2 = (age >= 18) ? "Adult" : "Minor";

Debug.Log(status2);
Output:
Adult
Example: Game Status
int health = 75;

string message = (health > 50) ? "Healthy" : "Damaged";
Debug.Log(message);

// Can use directly in method calls
Debug.Log((health == 0) ? "You died!" : "Still alive");
Output:
Healthy
Still alive

💡 When to Use Ternary:

  • ✅ Simple assignments based on one condition
  • ✅ Quick true/false decisions
  • ❌ Don't use for complex logic (hard to read)
  • ❌ Don't nest ternary operators
