💻 Lesson 3: Loops & Iteration

Learn how to repeat code efficiently with loops

🎯 What You'll Learn

  • ✅ Why loops are essential in programming
  • ✅ For loops - when you know how many times to repeat
  • ✅ While loops - when you repeat based on a condition
  • ✅ Do-while loops - when you need at least one execution
  • ✅ Break and continue statements
  • ✅ Nested loops for complex patterns

1. What is a Loop? 🔄

Concept:

Loop = repeat code multiple times without writing it over and over

Imagine you want to print "Hello" 100 times. Without loops, you'd need 100 lines of code. With loops, just 3 lines!

❌ Without Loops (Bad):

Without Loops - Repetitive Code
Debug.Log("Hello 1");
Debug.Log("Hello 2");
Debug.Log("Hello 3");
Debug.Log("Hello 4");
Debug.Log("Hello 5");
// ... 95 more lines 😱

✅ With Loops (Good):

With Loops - Clean Code
for (int i = 1; i <= 100; i++)
{
    Debug.Log("Hello " + i);
}
Output:
Hello 1
Hello 2
Hello 3
...
Hello 100

💡 Real-World Uses:

  • 🎮 Spawn 10 enemies in a game
  • 📊 Process all items in an inventory
  • 🔢 Calculate scores for all players
  • 🎨 Create grid patterns (tiles, buttons)
  • ⏱️ Game loop (runs every frame)

2. For Loop 🔢

Concept:

For loop = repeat code a specific number of times

Use when you know exactly how many times to loop

Syntax:

for (initialization; condition; increment)
{
    // code to repeat
}

📝 Basic For Loop:

Example: Count 1 to 5
for (int i = 1; i <= 5; i++)
{
    Debug.Log("Count: " + i);
}
Output:
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5

🧠 Understanding For Loop Parts:

  • int i = 1 → Initialization: Start at 1
  • i <= 5 → Condition: Continue while i is 5 or less
  • i++ → Increment: Add 1 after each loop

Flow:

  1. Set i = 1
  2. Check: is i <= 5? Yes → run code
  3. Increase i by 1 (i = 2)
  4. Check: is i <= 5? Yes → run code
  5. Repeat until i = 6 (condition false)

📝 Loop Starting from 0:

Example: Count 0 to 4 (Common in Programming)
for (int i = 0; i < 5; i++)
{
    Debug.Log("Index: " + i);
}
Output:
Index: 0
Index: 1
Index: 2
Index: 3
Index: 4
💡 Why Start at 0? Most programming (including arrays) starts counting at 0. Get used to thinking "0 to 4" = 5 items!

📝 Counting Backwards:

Example: Countdown
for (int i = 10; i >= 1; i--)
{
    Debug.Log(i);
}
Debug.Log("Liftoff! 🚀");
Output:
10
9
8
...
1
Liftoff! 🚀

📝 Different Increment Values:

Example: Count by 2s (Even Numbers)
for (int i = 0; i <= 10; i += 2)
{
    Debug.Log("Even: " + i);
}
Output:
Even: 0
Even: 2
Even: 4
Even: 6
Even: 8
Even: 10

📝 Real Game Example - Spawn Enemies:

Example: Spawn 5 Enemies
for (int i = 0; i < 5; i++)
{
    Debug.Log("Spawning enemy #" + (i+1));
    // In Unity, you would do:
    // Instantiate(enemyPrefab, position, rotation);
}
Output:
Spawning enemy #1
Spawning enemy #2
Spawning enemy #3
Spawning enemy #4
Spawning enemy #5

3. While Loop ⚡

Concept:

While loop = repeat code while a condition is true

Use when you don't know how many times to loop in advance

Syntax:

while (condition)
{
    // code to repeat
    // make sure condition eventually becomes false!
}

📝 Basic While Loop:

Example: Count with While
int count = 1;

while (count <= 5)
{
    Debug.Log("Count: " + count);
    count++; // IMPORTANT: increment to avoid infinite loop!
}
Output:
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
⚠️ Infinite Loop Warning: Always make sure the condition can become false! If you forget to increment, the loop runs forever and crashes your game!

📝 Game Example - Health System:

Example: Player Takes Damage Until Health is Low
int health = 100;
int turnCount = 0;

while (health > 0)
{
    turnCount++;
    health -= 25; // lose 25 health each turn
    Debug.Log("Turn " + turnCount + ": Health = " + health);
}

Debug.Log("Game Over! Survived " + turnCount + " turns");
Output:
Turn 1: Health = 75
Turn 2: Health = 50
Turn 3: Health = 25
Turn 4: Health = 0
Game Over! Survived 4 turns

📝 User Input Example:

Example: Keep Asking Until Correct Answer
string password = "";

while (password != "1234")
{
    Debug.Log("Enter password:");
    // In real Unity, you'd get input from UI
    password = "1234"; // simulating correct input
}

Debug.Log("Access granted! ✅");
Output:
Enter password:
Access granted! ✅

💡 For vs While:

Use For Loop When... Use While Loop When...
You know the exact number of iterations You don't know how many iterations
Example: "Loop 10 times" Example: "Loop until health = 0"
Example: Processing all items in a list Example: Waiting for player input

4. Do-While Loop 🔁

Concept:

Do-while loop = like while loop, but ALWAYS runs at least once

Checks condition AFTER running the code

Syntax:

do
{
    // code runs at least once
} while (condition);

📝 Do-While Example:

Example: Menu Loop
int choice = 0;

do
{
    Debug.Log("=== Game Menu ===");
    Debug.Log("1. Start Game");
    Debug.Log("2. Options");
    Debug.Log("3. Quit");
    Debug.Log("Your choice: ");
    
    // Simulate user selecting option 3
    choice = 3;
    
} while (choice != 3);

Debug.Log("Goodbye!");
Output:
=== Game Menu ===
1. Start Game
2. Options
3. Quit
Your choice:
Goodbye!

📝 Difference: While vs Do-While:

Example: While Loop (May Not Run)
int x = 10;

while (x < 5) // false from start
{
    Debug.Log("This never runs");
}
Output:
(no output - condition false from start)
Example: Do-While (Always Runs Once)
int x = 10;

do
{
    Debug.Log("This runs once even though x >= 5");
} while (x < 5); // false but already ran once
Output:
This runs once even though x >= 5
💡 When to use Do-While: When you need the code to run at least once before checking the condition (like showing a menu, asking for input, etc.)

5. Break and Continue ⏸️ ⏭️

Concept:

  • break; → Exit the loop immediately
  • continue; → Skip rest of current iteration, go to next

📝 Break Statement:

Example: Find and Stop
for (int i = 1; i <= 10; i++)
{
    if (i == 5)
    {
        Debug.Log("Found 5! Stopping...");
        break; // exit loop immediately
    }
    Debug.Log(i);
}
Output:
1
2
3
4
Found 5! Stopping...

📝 Continue Statement:

Example: Skip Even Numbers
for (int i = 1; i <= 10; i++)
{
    if (i % 2 == 0) // if even
    {
        continue; // skip rest, go to next iteration
    }
    Debug.Log("Odd number: " + i);
}
Output:
Odd number: 1
Odd number: 3
Odd number: 5
Odd number: 7
Odd number: 9

📝 Game Example - Search for Item:

Example: Find Weapon in Inventory
string[] inventory = {"potion", "key", "sword", "shield", "map"};

for (int i = 0; i < 5; i++)
{
    Debug.Log("Checking slot " + i + ": " + inventory[i]);
    
    if (inventory[i] == "sword")
    {
        Debug.Log("⚔️ Found sword at slot " + i + "!");
        break; // stop searching
    }
}
Output:
Checking slot 0: potion
Checking slot 1: key
Checking slot 2: sword
⚔️ Found sword at slot 2!

6. Nested Loops 📦📦

Concept:

Nested loop = a loop inside another loop

Used for working with 2D grids, tables, or multiple dimensions

📝 Basic Nested Loop:

Example: Multiplication Table
for (int i = 1; i <= 3; i++)
{
    for (int j = 1; j <= 3; j++)
    {
        Debug.Log(i + " x " + j + " = " + (i * j));
    }
    Debug.Log("---"); // separator
}
Output:
1 x 1 = 1
1 x 2 = 2
1 x 3 = 3
---
2 x 1 = 2
2 x 2 = 4
2 x 3 = 6
---
3 x 1 = 3
3 x 2 = 6
3 x 3 = 9
---

🧠 How Nested Loops Work:

For each iteration of the outer loop, the inner loop runs completely.

  1. Outer loop: i = 1
  2.     Inner loop runs: j = 1, 2, 3
  3. Outer loop: i = 2
  4.     Inner loop runs again: j = 1, 2, 3
  5. Outer loop: i = 3
  6.     Inner loop runs again: j = 1, 2, 3

📝 2D Grid Pattern:

Example: Create 3x3 Grid
for (int row = 0; row < 3; row++)
{
    string rowText = "";
    
    for (int col = 0; col < 3; col++)
    {
        rowText += "* ";
    }
    
    Debug.Log(rowText);
}
Output:
* * *
* * *
* * *

📝 Game Example - Spawn Enemy Grid:

Example: Spawn 3x3 Grid of Enemies
int rows = 3;
int cols = 3;

for (int row = 0; row < rows; row++)
{
    for (int col = 0; col < cols; col++)
    {
        Debug.Log("Spawning enemy at position (" + row + ", " + col + ")");
        // In Unity:
        // Vector3 pos = new Vector3(col * 2, 0, row * 2);
        // Instantiate(enemyPrefab, pos, Quaternion.identity);
    }
}
Output:
Spawning enemy at position (0, 0)
Spawning enemy at position (0, 1)
Spawning enemy at position (0, 2)
Spawning enemy at position (1, 0)
...
