1. What are Methods? 🔧

Concept:

Method/Function = a reusable block of code that performs a specific task

Think of it as a recipe: you write the recipe once, then use it many times

📝 Why Use Methods?

Benefits:

  • 🔄 Reusability: Write once, use many times
  • 📖 Readability: Code is easier to understand
  • 🐛 Debugging: Fix bugs in one place
  • 📦 Organization: Break large programs into smaller pieces

❌ Without Methods (Repetitive):

Bad: Repeating Code
Debug.Log("Player took damage!");
int playerHealth = 100;
playerHealth -= 10;
Debug.Log("Health: " + playerHealth);

// Same code again for enemy
Debug.Log("Enemy took damage!");
int enemyHealth = 50;
enemyHealth -= 10;
Debug.Log("Health: " + enemyHealth);

// ... more repetition ...

✅ With Methods (Clean & Reusable):

Good: Using Methods
void TakeDamage(string name, int health, int damage)
{
    Debug.Log(name + " took damage!");
    health -= damage;
    Debug.Log("Health: " + health);
}

// Now just call the method!
TakeDamage("Player", 100, 10);
TakeDamage("Enemy", 50, 10);

2. Creating Basic Methods 📝

Method Syntax:

returnType MethodName(parameters)
{
    // method body
    return value; // if returnType is not void
}

📝 Void Method (No Return Value):

Example: Simple Greeting
void SayHello()
{
    Debug.Log("Hello, Unity!");
}

// Call the method
SayHello();
SayHello();
SayHello();
Output:
Hello, Unity!
Hello, Unity!
Hello, Unity!

🧠 Understanding the Parts:

  • void = return type (void means "returns nothing")
  • SayHello = method name (use PascalCase)
  • () = parameters (empty means no parameters)
  • {} = method body (code that runs)

3. Parameters (Input Values) 📥

Concept:

Parameters = values you pass into a method to customize its behavior

📝 Method with One Parameter:

Example: Personalized Greeting
void GreetPlayer(string playerName)
{
    Debug.Log("Welcome, " + playerName + "!");
}

// Call with different names
GreetPlayer("Alex");
GreetPlayer("Jordan");
GreetPlayer("Sam");
Output:
Welcome, Alex!
Welcome, Jordan!
Welcome, Sam!

📝 Multiple Parameters:

Example: Calculate Damage
void DealDamage(string attacker, string target, int damage)
{
    Debug.Log(attacker + " attacks " + target + " for " + damage + " damage!");
}

DealDamage("Player", "Goblin", 25);
DealDamage("Dragon", "Player", 50);
Output:
Player attacks Goblin for 25 damage!
Dragon attacks Player for 50 damage!
💡 Parameter Order Matters: You must pass arguments in the same order as the parameters are defined!

4. Return Values (Output) 📤

Concept:

Return value = the result a method sends back after executing

Use return keyword to send back a value

📝 Returning a Value:

Example: Add Two Numbers
int Add(int a, int b)
{
    int result = a + b;
    return result;
}

int sum = Add(5, 3);
Debug.Log("Sum: " + sum);

// Can use directly
Debug.Log("10 + 20 = " + Add(10, 20));
Output:
Sum: 8
10 + 20 = 30

📝 Game Example - Calculate Experience:

Example: XP Calculation
int CalculateXP(int enemyLevel, bool isBoss)
{
    int baseXP = enemyLevel * 10;
    
    if (isBoss)
    {
        baseXP *= 5; // bosses give 5x XP
    }
    
    return baseXP;
}

int xp1 = CalculateXP(5, false);
Debug.Log("Normal enemy XP: " + xp1);

int xp2 = CalculateXP(10, true);
Debug.Log("Boss XP: " + xp2);
Output:
Normal enemy XP: 50
Boss XP: 500

📝 Returning Boolean (True/False):

Example: Check if Player Can Afford
bool CanAfford(int playerGold, int itemCost)
{
    return playerGold >= itemCost;
}

int gold = 100;

if (CanAfford(gold, 50))
{
    Debug.Log("Purchase successful!");
}
else
{
    Debug.Log("Not enough gold!");
}
Output:
Purchase successful!

5. Method Overloading 🔀

Concept:

Method overloading = multiple methods with the same name but different parameters

C# chooses the right method based on the arguments you pass

Example: Multiple Add Methods
// Add two integers
int Add(int a, int b)
{
    return a + b;
}

// Add three integers
int Add(int a, int b, int c)
{
    return a + b + c;
}

// Add two floats
float Add(float a, float b)
{
    return a + b;
}

Debug.Log(Add(5, 3)); // calls first method
Debug.Log(Add(5, 3, 2)); // calls second method
Debug.Log(Add(5.5f, 3.2f)); // calls third method
Output:
8
10
8.7
Example: Game Damage System
// Basic damage
void TakeDamage(int damage)
{
    Debug.Log("Took " + damage + " damage");
}

// Damage with type
void TakeDamage(int damage, string damageType)
{
    Debug.Log("Took " + damage + " " + damageType + " damage");
}

// Damage with type and attacker
void TakeDamage(int damage, string damageType, string attacker)
{
    Debug.Log(attacker + " dealt " + damage + " " + damageType + " damage");
}

TakeDamage(10);
TakeDamage(15, "Fire");
TakeDamage(20, "Ice", "Dragon");
Output:
Took 10 damage
Took 15 Fire damage
Dragon dealt 20 Ice damage

6. Variable Scope 🔭

Concept:

Scope = where a variable can be accessed in your code

📝 Local Scope (Inside Method):

Example: Local Variables
void MethodA()
{
    int localVar = 10; // only exists in MethodA
    Debug.Log("MethodA: " + localVar);
}

void MethodB()
{
    // localVar doesn't exist here!
    // Debug.Log(localVar); ← ERROR!
    Debug.Log("MethodB has no access to localVar");
}

📝 Class Scope (Available to All Methods):

Example: Class-Level Variables
public class Player : MonoBehaviour
{
    int health = 100; // class-level variable
    
    void TakeDamage(int damage)
    {
        health -= damage; // can access health
        Debug.Log("Health: " + health);
    }
    
    void Heal(int amount)
    {
        health += amount; // can also access health
        Debug.Log("Health: " + health);
    }
}

💡 Scope Rules:

  • Variables declared inside a method = local to that method only
  • Variables declared at class level = accessible by all methods in the class
  • Parameters = local to the method
  • Variables inside {} blocks (if, for, etc.) are local to that block
