1. What is Object-Oriented Programming? 🎭

Concept:

Object-Oriented Programming (OOP) = organizing code around objects that represent real-world things

Instead of writing everything in one place, you create classes (blueprints) and objects (instances)

💡 Real-World Analogy:

Car Blueprint (Class):

  • 🎨 Defines properties: color, speed, fuel
  • ⚙️ Defines actions: accelerate(), brake(), honk()

Actual Cars (Objects):

  • 🚗 My red car (speed: 100 km/h)
  • 🚙 Your blue car (speed: 120 km/h)
  • 🚕 Friend's yellow car (speed: 90 km/h)

Same blueprint, different instances with different values!

🎮 Game Example:

Enemy Class (Blueprint):

  • Properties: health, damage, speed
  • Methods: Attack(), TakeDamage(), Move()

Enemy Objects:

  • 👹 Goblin (health: 50, damage: 10)
  • 🐉 Dragon (health: 500, damage: 100)
  • 🧟 Zombie (health: 30, damage: 5)

2. Classes and Objects 🏗️

Concept:

  • Class = blueprint/template that defines properties and behaviors
  • Object = specific instance created from a class

📝 Creating Your First Class:

Example: Player Class
public class Player
{
    // Fields (variables inside class)
    public string playerName;
    public int health;
    public int level;
    
    // Method
    public void ShowInfo()
    {
        Debug.Log("Name: " + playerName);
        Debug.Log("Health: " + health);
        Debug.Log("Level: " + level);
    }
}

📝 Creating Objects from the Class:

Example: Create Player Objects
// Create first player
Player player1 = new Player();
player1.playerName = "Alex";
player1.health = 100;
player1.level = 5;

// Create second player
Player player2 = new Player();
player2.playerName = "Jordan";
player2.health = 80;
player2.level = 3;

// Use the objects
player1.ShowInfo();
player2.ShowInfo();
Output:
Name: Alex
Health: 100
Level: 5
Name: Jordan
Health: 80
Level: 3

🧠 Understanding the Syntax:

  • public class Player → defines the class
  • Player player1 = new Player(); → creates an object
  • new → keyword to create a new instance
  • player1.health → access field using dot notation
  • player1.ShowInfo() → call method on object

3. Constructors 🔧

Concept:

Constructor = special method that runs when you create an object

Used to initialize values automatically

📝 Class with Constructor:

Example: Enemy Class with Constructor
public class Enemy
{
    public string name;
    public int health;
    public int damage;
    
    // Constructor (same name as class, no return type)
    public Enemy(string enemyName, int enemyHealth, int enemyDamage)
    {
        name = enemyName;
        health = enemyHealth;
        damage = enemyDamage;
        Debug.Log(name + " spawned!");
    }
    
    public void Attack()
    {
        Debug.Log(name + " attacks for " + damage + " damage!");
    }
}

📝 Using the Constructor:

Example: Create Enemies Easily
// Much cleaner! Values set immediately
Enemy goblin = new Enemy("Goblin", 50, 10);
Enemy dragon = new Enemy("Dragon", 500, 100);
Enemy zombie = new Enemy("Zombie", 30, 5);

goblin.Attack();
dragon.Attack();
Output:
Goblin spawned!
Dragon spawned!
Zombie spawned!
Goblin attacks for 10 damage!
Dragon attacks for 100 damage!

📝 Default Constructor:

Example: Multiple Constructors
public class Weapon
{
    public string name;
    public int damage;
    
    // Default constructor
    public Weapon()
    {
        name = "Fists";
        damage = 5;
    }
    
    // Constructor with parameters
    public Weapon(string weaponName, int weaponDamage)
    {
        name = weaponName;
        damage = weaponDamage;
    }
}

Weapon defaultWeapon = new Weapon(); // Uses default
Weapon sword = new Weapon("Sword", 50); // Uses parameterized

4. Encapsulation (Data Hiding) 🔒

Concept:

Encapsulation = hiding internal data and controlling access through methods

Use private fields and public properties/methods

Access Modifiers:

Modifier Meaning When to Use
public Accessible from anywhere Methods, properties you want others to use
private Only accessible inside the class Internal data, helper methods
protected Accessible in class and subclasses Data for inheritance

📝 Without Encapsulation (Bad):

Bad: Direct Access
public class BankAccount
{
    public int balance; // Anyone can modify!
}

BankAccount account = new BankAccount();
account.balance = 1000;
account.balance = -500; // ❌ Negative balance! Bug!

📝 With Encapsulation (Good):

Good: Controlled Access
public class BankAccount
{
    private int balance; // Hidden
    
    public void Deposit(int amount)
    {
        if (amount > 0)
        {
            balance += amount;
            Debug.Log("Deposited: " + amount);
        }
    }
    
    public bool Withdraw(int amount)
    {
        if (amount > 0 && balance >= amount)
        {
            balance -= amount;
            return true;
        }
        Debug.Log("Insufficient funds");
        return false;
    }
    
    public int GetBalance()
    {
        return balance;
    }
}

📝 Properties (C# Shortcut):

Example: Using Properties
public class Player
{
    private int health; // Private field
    
    // Public property
    public int Health
    {
        get { return health; } // Read
        set
        {
            if (value >= 0 && value <= 100) // Validate
            {
                health = value;
            }
        }
    }
}

Player player = new Player();
player.Health = 80; // Uses set
player.Health = 150; // Rejected (over 100)
Debug.Log(player.Health); // Uses get

5. Inheritance (Code Reuse) 🧬

Concept:

Inheritance = creating new classes based on existing ones

Child class inherits all properties and methods from parent class

📝 Base Class (Parent):

Example: Character Base Class
public class Character
{
    public string name;
    public int health;
    public int damage;
    
    public void TakeDamage(int amount)
    {
        health -= amount;
        Debug.Log(name + " took " + amount + " damage. Health: " + health);
    }
    
    public void Attack(Character target)
    {
        Debug.Log(name + " attacks " + target.name);
        target.TakeDamage(damage);
    }
}

📝 Derived Classes (Children):

Example: Specialized Character Classes
// Warrior inherits from Character
public class Warrior : Character
{
    public int armor;
    
    public void Shield()
    {
        Debug.Log(name + " raises shield! Armor: " + armor);
    }
}

// Mage inherits from Character
public class Mage : Character
{
    public int mana;
    
    public void CastSpell()
    {
        if (mana >= 10)
        {
            mana -= 10;
            Debug.Log(name + " casts fireball!");
        }
    }
}

📝 Using Inherited Classes:

Example: Warrior and Mage Objects
Warrior warrior = new Warrior();
warrior.name = "Conan";
warrior.health = 150;
warrior.damage = 30;
warrior.armor = 50;

Mage mage = new Mage();
mage.name = "Gandalf";
mage.health = 80;
mage.damage = 50;
mage.mana = 100;

// Use inherited methods
warrior.Attack(mage); // From Character
warrior.Shield(); // Warrior-specific

mage.CastSpell(); // Mage-specific
Output:
Conan attacks Gandalf
Gandalf took 30 damage. Health: 50
Conan raises shield! Armor: 50
Gandalf casts fireball!

💡 Inheritance Benefits:

  • ✅ **Code reuse** - don't repeat common code
  • ✅ **Organization** - clear hierarchy
  • ✅ **Maintainability** - fix/change in one place
  • ✅ **Extensibility** - easy to add new types

6. Polymorphism (Many Forms) 🎭

Concept:

Polymorphism = same method name, different implementations in child classes

Use virtual in parent, override in child

📝 Virtual Method in Base Class:

Example: Base Animal Class
public class Animal
{
    public string name;
    
    // Virtual method (can be overridden)
    public virtual void MakeSound()
    {
        Debug.Log("Some generic animal sound");
    }
}

📝 Override in Derived Classes:

Example: Different Animal Sounds
public class Dog : Animal
{
    public override void MakeSound()
    {
        Debug.Log(name + " says: Woof! 🐕");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Debug.Log(name + " says: Meow! 🐈");
    }
}

public class Cow : Animal
{
    public override void MakeSound()
    {
        Debug.Log(name + " says: Moo! 🐄");
    }
}

📝 The Magic of Polymorphism:

Example: One Array, Many Types
// Create different animals
Animal[] animals = new Animal[3];

animals[0] = new Dog() { name = "Buddy" };
animals[1] = new Cat() { name = "Whiskers" };
animals[2] = new Cow() { name = "Bessie" };

// Same method call, different behavior!
foreach (Animal animal in animals)
{
    animal.MakeSound(); // Calls the correct override
}
Output:
Buddy says: Woof! 🐕
Whiskers says: Meow! 🐈
Bessie says: Moo! 🐄

📝 Game Example - Enemy AI:

Example: Different Enemy Behaviors
public class Enemy
{
    public string name;
    
    public virtual void Attack()
    {
        Debug.Log(name + " attacks!");
    }
}

public class MeleeEnemy : Enemy
{
    public override void Attack()
    {
        Debug.Log(name + " swings sword! ⚔️");
    }
}

public class RangedEnemy : Enemy
{
    public override void Attack()
    {
        Debug.Log(name + " shoots arrow! 🏹");
    }
}

public class MagicEnemy : Enemy
{
    public override void Attack()
    {
        Debug.Log(name + " casts spell! 🔮");
    }
}

// Game loop
List<Enemy> enemies = new List<Enemy>();
enemies.Add(new MeleeEnemy() { name = "Knight" });
enemies.Add(new RangedEnemy() { name = "Archer" });
enemies.Add(new MagicEnemy() { name = "Wizard" });

foreach (Enemy enemy in enemies)
{
    enemy.Attack(); // Each enemy attacks differently!
}
Output:
Knight swings sword! ⚔️
Archer shoots arrow! 🏹
Wizard casts spell! 🔮
