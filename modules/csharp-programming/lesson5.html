💻 Lesson 5: Arrays & Collections

Learn to store and manage multiple values efficiently

🎯 What You'll Learn

  • ✅ Store multiple values in arrays
  • ✅ Access and modify array elements
  • ✅ Loop through arrays efficiently
  • ✅ Create 2D arrays for grids and boards
  • ✅ Use Lists for dynamic collections
  • ✅ Build game systems (inventory, scores, enemies)

1. What are Arrays? 📦

Concept:

Array = a container that holds multiple values of the same type

Instead of creating variables like score1, score2, score3..., use one array!

❌ Without Arrays:

Bad: Individual Variables
int score1 = 100;
int score2 = 85;
int score3 = 92;
int score4 = 78;
int score5 = 95;
// What if you need 100 scores? 😱

✅ With Arrays:

Good: Using Arrays
int[] scores = {100, 85, 92, 78, 95};
// All scores in one variable! Access by index

💡 Real-World Analogy:

Think of an array like an apartment building:

  • 🏢 The building = the array
  • 🚪 Room numbers (0, 1, 2, 3...) = indices
  • 👤 People in rooms = values stored
  • 📍 Address "Room 3" = array[3]

2. Creating and Initializing Arrays 🔨

📝 Method 1: Declare with Values:

Example: Initialize with Values
string[] weapons = {"Sword", "Bow", "Staff", "Axe"};
int[] levels = {1, 5, 10, 15, 20};
float[] prices = {9.99f, 19.99f, 4.99f};

📝 Method 2: Declare Size, Then Assign:

Example: Create Empty Array
int[] playerScores = new int[5]; // creates array for 5 integers

// Assign values later
playerScores[0] = 100;
playerScores[1] = 85;
playerScores[2] = 92;
playerScores[3] = 78;
playerScores[4] = 95;

🧠 Understanding Array Syntax:

  • int[] = type of array (int array)
  • new int[5] = creates array with 5 slots
  • Indices start at 0 (not 1!)
  • Last index = length - 1
⚠️ Important: Array size is FIXED once created. You cannot add more elements later!

3. Accessing Array Elements 🎯

Concept:

Access elements using index in square brackets: arrayName[index]

Remember: Indices start at 0!

Example: Accessing Elements
string[] enemies = {"Goblin", "Orc", "Dragon", "Zombie"};

Debug.Log(enemies[0]); // First element
Debug.Log(enemies[1]); // Second element
Debug.Log(enemies[2]); // Third element
Debug.Log(enemies[3]); // Fourth element
// Debug.Log(enemies[4]); ← ERROR! Out of bounds
Output:
Goblin
Orc
Dragon
Zombie

📝 Modifying Elements:

Example: Change Values
int[] health = {100, 80, 60};
Debug.Log("Before: " + health[1]);

health[1] = 90; // change second element
Debug.Log("After: " + health[1]);
Output:
Before: 80
After: 90

📝 Array Length Property:

Example: Get Array Size
string[] items = {"Potion", "Key", "Map", "Coin", "Gem"};

Debug.Log("Inventory has " + items.Length + " items");
Debug.Log("Last item: " + items[items.Length - 1]);
Output:
Inventory has 5 items
Last item: Gem

4. Looping Through Arrays 🔄

📝 Method 1: For Loop (Traditional):

Example: For Loop with Index
int[] scores = {85, 92, 78, 95, 88};

for (int i = 0; i < scores.Length; i++)
{
    Debug.Log("Player " + (i+1) + ": " + scores[i] + " points");
}
Output:
Player 1: 85 points
Player 2: 92 points
Player 3: 78 points
Player 4: 95 points
Player 5: 88 points

📝 Method 2: Foreach Loop (Simpler):

Example: Foreach Loop
string[] weapons = {"Sword", "Bow", "Staff"};

foreach (string weapon in weapons)
{
    Debug.Log("⚔️ Equipped: " + weapon);
}
Output:
⚔️ Equipped: Sword
⚔️ Equipped: Bow
⚔️ Equipped: Staff

💡 For vs Foreach:

For Loop Foreach Loop
Need index position Just need values
Can modify elements Read-only
More flexible Simpler syntax

📝 Calculate Array Sum:

Example: Sum All Elements
int[] numbers = {10, 20, 30, 40, 50};
int total = 0;

foreach (int num in numbers)
{
    total += num;
}

Debug.Log("Total: " + total);
Debug.Log("Average: " + (total / numbers.Length));
Output:
Total: 150
Average: 30

5. Multi-Dimensional Arrays (2D) 🎮

Concept:

2D Array = array of arrays (like a grid or table)

Perfect for: game boards, tile maps, grids

Example: 3x3 Game Board
int[,] board =
{
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

Debug.Log("Center tile: " + board[1, 1]);
Debug.Log("Top-left: " + board[0, 0]);
Debug.Log("Bottom-right: " + board[2, 2]);
Output:
Center tile: 5
Top-left: 1
Bottom-right: 9

📝 Loop Through 2D Array:

Example: Print Grid
string[,] dungeon =
{
    {"🚪", "⬜", "⬜"},
    {"⬜", "👹", "⬜"},
    {"⬜", "⬜", "🏆"}
};

for (int row = 0; row < 3; row++)
{
    string line = "";
    for (int col = 0; col < 3; col++)
    {
        line += dungeon[row, col] + " ";
    }
    Debug.Log(line);
}
Output:
🚪 ⬜ ⬜
⬜ 👹 ⬜
⬜ ⬜ 🏆

6. Lists (Dynamic Collections) 📋

Concept:

List = like an array, but can grow and shrink

Use System.Collections.Generic.List<T>

💡 Array vs List:

Array List
Fixed size Dynamic size
Faster More flexible
Use when size is known Use when size changes

📝 Creating and Using Lists:

Example: Inventory System
using System.Collections.Generic;

List<string> inventory = new List<string>();

// Add items
inventory.Add("Sword");
inventory.Add("Potion");
inventory.Add("Key");
Debug.Log("Items: " + inventory.Count);

// Remove item
inventory.Remove("Potion");
Debug.Log("Items after removal: " + inventory.Count);

// Check if contains
if (inventory.Contains("Sword"))
{
    Debug.Log("Has sword!");
}
Output:
Items: 3
Items after removal: 2
Has sword!

📝 List Methods:

Example: Common List Operations
List<int> scores = new List<int>() {85, 92, 78};

scores.Add(95); // Add to end
scores.Insert(0, 100); // Insert at index 0
scores.RemoveAt(2); // Remove at index 2
scores.Clear(); // Remove all
Debug.Log("Count: " + scores.Count);
Output:
Count: 0

📝 Game Example - Enemy Spawner:

Example: Dynamic Enemy List
List<string> activeEnemies = new List<string>();

// Spawn enemies
activeEnemies.Add("Goblin");
activeEnemies.Add("Orc");
activeEnemies.Add("Troll");
Debug.Log("Enemies spawned: " + activeEnemies.Count);

// Enemy defeated
activeEnemies.Remove("Orc");
Debug.Log("Enemies remaining: " + activeEnemies.Count);

// List all enemies
foreach (string enemy in activeEnemies)
{
    Debug.Log("⚔️ Fighting: " + enemy);
}
Output:
Enemies spawned: 3
Enemies remaining: 2
⚔️ Fighting: Goblin
⚔️ Fighting: Troll
