1. Finding Free Game Assets on itch.io 🎮

Concept:

itch.io is a popular platform for indie game developers where you can find thousands of free and paid game assets, publish your games, and connect with the game dev community.

How to Find Assets:

📋 Steps:

  1. Visit itch.io in your web browser
  2. Navigate to Game Assets section
  3. Filter by Free to find free resources
  4. Search for the type of assets you need (e.g., "pixel art warrior")
  5. Download the asset pack (usually a .zip file)
Example: Warrior Sprite Pack
Search Term: "pixel art warrior"
Pack Name: "Warrior Pack"
Contents: Sprite sheets with idle, run, attack animations
Format: .png files in a .zip archive
💡 Tip: Always check the license! Most free assets allow use in personal and commercial projects, but always read the terms. Give credit to the artist when possible!

2. Importing Sprites into Unity 📦

Concept:

After downloading sprites, you need to import them into Unity and configure them properly to avoid blurry or pixelated graphics.

Import Process:

📋 Steps:

  1. Extract the .zip file you downloaded from itch.io
  2. In Unity, drag the sprite sheets into your Assets folder
  3. Select the imported sprite in the Project window
  4. In the Inspector, configure the following settings:

Critical Import Settings:

Sprite Import Configuration
Texture Type: Sprite (2D and UI)
Sprite Mode: Multiple
Pixels Per Unit: 16
Compression: None
Filter Mode: Point (no filter)
Max Size: Keep default (usually 2048)

⚙️ Why These Settings Matter:

  • Pixels Per Unit (16): Matches pixel art scale - means 16 pixels = 1 Unity unit
  • Compression: None: Prevents artifacts and keeps pixel art crisp
  • Filter Mode: Point: Keeps sharp pixel edges (no blurring)
  • Sprite Mode: Multiple: Allows slicing sprite sheets into individual sprites
⚠️ Important: Always click Apply after changing settings! Unity won't save your changes otherwise.

3. Separating Logic and Visuals 🏗️

Concept:

A best practice in game development is to separate your game logic from visual elements using a parent-child GameObject structure.

Why Separate?

✅ Benefits:

  • Keep physics and logic on the parent GameObject
  • Keep visuals (sprites, animations) on child GameObjects
  • Easier to swap out visual elements without breaking code
  • Better organization and cleaner hierarchy
  • Allows independent scaling and rotation of visuals

Implementation:

GameObject Structure
Player (Parent GameObject)
  ├── Rigidbody2D
  ├── Collider2D
  ├── PlayerController script
  └── Graphics (Child GameObject)
       ├── SpriteRenderer
       └── Animator

📋 Setup Steps:

  1. Keep your existing Player GameObject with all scripts and physics
  2. Right-click on Player → Create Empty → Name it "Graphics"
  3. On Graphics: Add Component → Sprite Renderer
  4. Remove the Sprite Renderer from the parent Player object
  5. Drag your sprite onto the Graphics object's Sprite Renderer
💡 Pro Tip: This pattern makes it easy to add effects like sprite flipping, squash/stretch, or visual-only rotations without affecting your physics!

4. Understanding Pivot Points 🎯

Concept:

A pivot point is the reference point for a sprite - where rotation, scaling, and positioning happen from. Think of it like a door hinge!

The Door Hinge Analogy:

Pivot Point Like a Door Hinge
🚪 A door rotates around its HINGE, not its center.

If the hinge is on the LEFT:
  → Door swings from the left edge

If the hinge is in the CENTER:
  → Door would spin in place (weird!)

Similarly, a sprite's PIVOT is where it rotates and scales from.

Common Pivot Positions:

📍 Pivot Options:

  • Bottom Center: Good for characters (ground contact point)
  • Center: Good for projectiles or floating objects
  • Custom: Set manually during sprite slicing
💡 Why It Matters: If your character sprite's pivot is at the top instead of the bottom, when you place it at Y=0, only its top will be visible! The pivot determines the GameObject's position reference.

5. Slicing Sprite Sheets ✂️

Concept:

A sprite sheet contains multiple animation frames in a single image file. Unity's Sprite Editor lets you slice it into individual sprites for animation.

Using the Sprite Editor:

📋 Steps:

  1. Select your sprite sheet in the Project window
  2. Make sure Sprite Mode is set to "Multiple"
  3. Click Sprite Editor button in Inspector
  4. In Sprite Editor, click Slice dropdown
  5. Choose Grid By Cell Size
  6. Enter the pixel dimensions of each frame
  7. Click Slice, then Apply

Example: Warrior Sprite Sheet:

Sprite Slice Settings
Type: Grid By Cell Size
Pixel Size:
  X: 69
  Y: 44

Pivot: Bottom (or Bottom Center)

Result: Individual sprites appear as children in Project window

🎨 After Slicing:

  • Click the arrow next to your sprite sheet to see all individual frames
  • Each frame is named automatically (e.g., warrior_0, warrior_1, etc.)
  • You can now drag individual frames into your scene or use them in animations
💡 Finding Cell Size: If you don't know the cell size, open the sprite sheet in an image editor and measure one frame's width and height in pixels.

6. Pivot vs Center Transform Mode 🔄

Concept:

Unity's Scene view has two transform tool modes: Pivot and Center. This affects where the transform gizmo appears.

Transform Modes:

Pivot vs Center
Pivot Mode:
  → Gizmo appears at the sprite's actual pivot point
  → Shows where rotation/scaling happens from
  → Good for precise positioning

Center Mode:
  → Gizmo appears at the geometric center
  → Ignores the sprite's pivot setting
  → Good for centering objects visually

How to Switch:

🔧 Location:

Look at the top-left of the Unity Scene view toolbar. You'll see two buttons:

  • Left button: Toggle between Pivot / Center
  • Right button: Toggle between Local / Global rotation
💡 Tip: When positioning your Graphics child object, use Pivot mode to see exactly where the sprite's pivot is relative to the parent Player object!

7. Updating the Collider 🛡️

Concept:

When you replace a circle placeholder with a character sprite, you should update the collider to match the new shape for better physics and collision detection.

Changing Collider Type:

📋 Steps:

  1. Select your Player GameObject
  2. In Inspector, find the Circle Collider 2D component
  3. Click the three dots (⋮) → Remove Component
  4. Click Add Component → Search "Capsule"
  5. Add Capsule Collider 2D
  6. Adjust the size to fit your character sprite

Why Capsule Collider?

Collider Comparison
Circle Collider 2D:
  ✅ Good for: Rolling objects, projectiles
  ❌ Bad for: Humanoid characters (catches on edges)

Capsule Collider 2D:
  ✅ Good for: Characters (smooth ledge climbing)
  ✅ Rounded top/bottom prevents snagging
  ✅ Flat sides for stable standing
💡 Pro Tip: Make the capsule slightly smaller than your sprite so the character doesn't appear to "float" when standing next to walls or on platforms.

8. Fixing Common Issues 🔧

Concept:

When adding visuals to your player, you might encounter issues like unwanted rotation. Here's how to fix the most common problems.

Issue: Player Character Rotating/Falling Over

Problem
When you move, your character tips over or rotates!
This happens because Rigidbody2D allows rotation by default.

Solution: Freeze Rotation

📋 Steps:

  1. Select your Player GameObject
  2. Find the Rigidbody2D component in Inspector
  3. Expand the Constraints section
  4. Check the box for Freeze Rotation Z
Rigidbody2D Constraints
Freeze Position:
  ☐ X
  ☐ Y

Freeze Rotation:
  ☑ Z ← Check this!

Other Common Issues:

🐛 Common Problems & Fixes:

  • Blurry sprites: Set Filter Mode to "Point (no filter)"
  • Wrong scale: Adjust Pixels Per Unit (try 16, 32, or 64)
  • Sprite offset: Check Graphics child position (should be 0, 0, 0 or slight Y offset)
  • Animation not playing: Make sure Animator component is on Graphics child, not parent

9. Creating Your First Animation 🎬

Concept:

Unity's Animation window lets you create animations by arranging sprite frames over time. An idle animation loops continuously when the player is standing still.

Creating an Idle Animation:

📋 Steps:

  1. Select your Graphics child GameObject
  2. Open Window → Animation → Animation
  3. Click Create button in Animation window
  4. Name it "playerIdle" and save in your project
  5. This automatically creates an Animator Controller too

Adding Frames to the Animation:

📋 Steps:

  1. In Project window, expand your sliced sprite sheet
  2. Select the first 6 idle frames (e.g., warrior_0 through warrior_5)
  3. Drag them into the Animation window timeline
  4. Unity automatically spaces them out over time

Adjusting Animation Speed:

Sample Rate Settings
In Animation window, find "Sample Rate" (usually top-left)

Default: 60 (very fast!)
For Idle Animation: 10

Sample Rate = Frames Per Second (FPS)
Lower = Slower animation
Higher = Faster animation

🎨 Animation Best Practices:

  • Idle animations: 6-12 FPS (slow and subtle)
  • Run animations: 10-15 FPS (moderate speed)
  • Attack animations: 15-24 FPS (fast and snappy)
  • Make sure Loop is enabled for idle animations
💡 Tip: Press the Play button in the Animation window (not the main Play button) to preview your animation without entering Play Mode!

10. Understanding the Animator 🎭

Concept:

The Animator system has two parts: the Animator component (on your GameObject) and the Animator Controller (an asset that holds animation states and transitions).

Animator Components:

Animator System Structure
Animator Component: (On Graphics GameObject)
  → References an Animator Controller
  → Controls which animation plays
  → Updated every frame

Animator Controller: (Asset in Project)
  → Contains animation states (Idle, Run, Jump, etc.)
  → Defines transitions between states
  → Reusable across multiple GameObjects

What Happened Automatically:

✨ When you created your first animation:

  • Unity added an Animator component to your Graphics object
  • Unity created an Animator Controller asset
  • Your "playerIdle" animation was added as the default state
  • The controller was automatically assigned to the Animator component

Viewing the Animator Controller:

📋 Steps:

  1. Open Window → Animation → Animator
  2. Select your Graphics GameObject
  3. You'll see your animation states as boxes
  4. Orange state = Default/Entry state (plays first)
Animator Controller View
┌──────────┐
│ Entry │──→ ┌─────────────┐
└──────────┘ │ playerIdle │ ← Orange (default)
                 └─────────────┘
                     ↻ (loops)

11. Project Organization 📁

Concept:

As your project grows, keeping files organized is crucial. Create dedicated folders for different asset types to stay organized.

Recommended Folder Structure:

Assets Organization
Assets/
├── Animations/ ← Animation clips (.anim)
│ ├── playerIdle.anim
│ └── playerRun.anim
├── Animators/ ← Animator Controllers
│ └── PlayerAnimator.controller
├── Scripts/
│ └── PlayerController.cs
├── Sprites/
│ ├── Characters/
│ │ └── warrior.png
│ └── Environment/
├── Scenes/
│ └── MainScene.unity
└── Prefabs/

Creating an Animations Folder:

📋 Steps:

  1. In Project window, right-click in Assets
  2. Create → Folder
  3. Name it "Animations"
  4. Drag your playerIdle.anim file into this folder
  5. Create another folder called "Animators" for controllers
💡 Pro Tip: Organize as you go! It's much harder to reorganize a messy project later. Create folders BEFORE you have hundreds of files.
