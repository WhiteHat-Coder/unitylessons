🎯 Lesson 7: Collision Detection & Ground Checking

Learn how to detect the ground beneath your character using raycasts and prevent endless jumping

🎯 What You'll Learn

  • ✅ Understand why collision detection is essential for realistic gameplay
  • ✅ Master the Physics2D.Raycast system for ground detection
  • ✅ Use Gizmos for visual debugging in the Scene view
  • ✅ Implement Layer Masks to detect specific objects
  • ✅ Create proper jump restrictions based on ground state
  • ✅ Organize inspector variables with Headers
  • ✅ Fine-tune physics parameters for better game feel

1. The Problem: Endless Jumping 🚫

Concept:

Without collision detection, players can jump infinitely while already in the air. This breaks the physics realism of your game. We need a way to detect if the character is standing on the ground before allowing them to jump.

🎮 Why This Matters

  • Game Feel: Realistic physics make your game feel professional
  • Fair Gameplay: Players shouldn't be able to fly by spamming jump
  • Platformer Standard: Ground detection is fundamental in 2D games
  • Design Control: You control when and how players can jump
💡 Tip: Collision detection might seem complex at first, but with practice it becomes second nature. Don't worry if you don't understand everything immediately—we'll be using this technique frequently!

2. Organizing Inspector with Headers 📋

Concept:

As your scripts grow more complex, organization becomes crucial. Unity's [Header] attribute creates visual sections in the Inspector, making it easier to find and adjust related variables.

Adding Headers to Your Script

Header Organization
[Header("Movement Details")]
[SerializeField] private float moveSpeed = 5f;
[SerializeField] private float jumpForce = 12f;

[Header("Collision Details")]
[SerializeField] private float groundCheckDistance;
[SerializeField] private LayerMask whatIsGround;

✨ Benefits of Headers

  • Visual Separation: Creates clear sections in the Inspector
  • Better Workflow: Quickly find the variables you need
  • Team Communication: Other developers understand your code structure
  • Professional Practice: Industry-standard code organization

3. Ground Check Distance Variable 📏

Concept:

The Ground Check Distance determines how far down from the character's center we'll cast a "laser" (raycast) to detect the ground. Think of it as the length of an invisible measuring stick pointing downward.

Creating the Variable

Ground Check Setup
[Header("Collision Details")]
[SerializeField] private float groundCheckDistance;

🎯 How It Works

  • Origin Point: Starts from the character's transform.position (center)
  • Direction: Shoots straight down (negative Y direction)
  • Length: The groundCheckDistance value determines how far
  • Detection: If it hits something tagged as "ground," we're standing on ground
⚙️ Typical Value: For most 2D characters, a groundCheckDistance of 0.4 to 1.4 works well. You'll fine-tune this based on your character's size and collider shape.

4. Visual Debugging with OnDrawGizmos 👁️

Concept:

Gizmos are visual debugging tools that draw shapes, lines, and icons in the Scene view (not visible to players). They help you visualize what your code is doing, making it much easier to debug and fine-tune collision detection.

Drawing the Ground Check Line

OnDrawGizmos Method
private void OnDrawGizmos()
{
    Gizmos.DrawLine(transform.position,
        transform.position + new Vector3(0, -groundCheckDistance, 0));
}

🔍 Understanding the Code

  • OnDrawGizmos(): Special Unity method that runs in the editor
  • Gizmos.DrawLine: Draws a line between two points
  • First Point: transform.position (character center)
  • Second Point: transform.position + Vector3(0, -groundCheckDistance, 0)
  • Vector3 Math: Adding (0, -negative value, 0) moves the point downward
🎨 Important: Gizmos ONLY draw visual lines—they don't do any actual physics detection! You need both the gizmo (to see it) and the raycast (to detect collisions) with the same parameters.

Adjusting in the Inspector

📊 Fine-Tuning Process

  • Select your character in the Scene view and zoom in
  • Look for the line drawn below the character
  • Adjust groundCheckDistance value in the Inspector
  • Watch the line extend or shrink in real-time
  • The line should reach slightly into the ground platform
Typical Values to Try
groundCheckDistance = 30 → Too long (way too far down)
groundCheckDistance = 1.30 → Still too long
groundCheckDistance = 0.4 → Good for most characters
groundCheckDistance = 1.4 → Good if character is taller

5. Adjusting Collider Offset 🎯

Concept:

Sometimes your character's visual sprite and collider center don't align perfectly. The Collider Offset lets you adjust the collider's position relative to the sprite, ensuring accurate ground detection.

Fixing Offset Issues
// In the Capsule Collider 2D component:
Offset X: 0
Offset Y: 0.3

// This centers the collider properly on the character sprite

🔧 When to Adjust Offset

  • Visual Misalignment: Ground check line doesn't appear centered
  • Collision Problems: Character sinks into or floats above ground
  • Sprite Pivot Issues: Your sprite's pivot point isn't at the center
  • Trial and Error: Test values between 0 and 0.3 for Y offset

6. Creating the HandleCollision Method 🛠️

Concept:

Good code organization means separating concerns. The HandleCollision() method will be responsible for all ground detection logic, keeping your Update() method clean and readable.

Method Structure

HandleCollision Setup
private void HandleCollision()
{
    // Ground detection logic will go here
}

private void Update()
{
    HandleCollision(); // Call first, before movement
    HandleInput();
    HandleMovement();
}

📌 Why Call It First?

  • Fresh Data: Check ground state before processing input
  • Prevents Bugs: Ensures ground state is current before jump attempts
  • Logical Flow: Detect → Input → React is a clear sequence
  • Frame Accuracy: Ground state updates every frame before use

7. The isGrounded Boolean 🚦

Concept:

We need a boolean variable (true/false) to store whether the character is currently touching the ground. This simple flag will control when jumping is allowed.

Boolean Declaration
private bool isGrounded;

🎮 How It's Used

  • True: Character is standing on ground → Can jump
  • False: Character is in the air → Cannot jump
  • Updated Every Frame: Constantly refreshed by HandleCollision()
  • Simple Gate: Single variable controls jump permission
🔒 Private Variable: We don't need to expose this in the Inspector initially. We know it's working when our jump restrictions work correctly.

8. Physics2D.Raycast - The Detection System ⚡

Concept:

Physics2D.Raycast is Unity's built-in function that shoots an invisible ray through your game world and returns true if it hits something, or false if it hits nothing. Think of it as an invisible laser pointer that can detect objects.

The Complete Raycast

Full Implementation
private void HandleCollision()
{
    isGrounded = Physics2D.Raycast(
        transform.position, // Origin: where to start the ray
        Vector2.down, // Direction: which way to shoot
        groundCheckDistance, // Distance: how far to check
        whatIsGround // LayerMask: what to detect
    );
}

🎯 Breaking Down the Parameters

  • Origin (transform.position): Starting point of the ray - the character's center
  • Direction (Vector2.down): Which way to shoot - downward (0, -1)
  • Distance (groundCheckDistance): How far the ray travels - our custom value
  • LayerMask (whatIsGround): Filter to detect only specific objects

Understanding Vector2.down

Direction Vectors
Vector2.down = (0, -1) // Straight down
Vector2.up = (0, 1) // Straight up
Vector2.left = (-1, 0) // Straight left
Vector2.right = (1, 0) // Straight right
🎓 Pro Tip: You can shoot raycasts in any direction by changing the Vector2 parameter. Want to detect walls? Use Vector2.right or Vector2.left!

9. Layer Masks and Ground Layer Setup 🏷️

Concept:

Without a LayerMask, your raycast would detect EVERYTHING—players, enemies, coins, props. Layer Masks act as filters, allowing you to specify exactly which types of objects you want to detect.

Creating the LayerMask Variable

LayerMask Declaration
[Header("Collision Details")]
[SerializeField] private float groundCheckDistance;
[SerializeField] private LayerMask whatIsGround;

🎯 Setup Steps in Unity

  • Step 1: In Inspector, go to "Layer" dropdown (top right)
  • Step 2: Click "Add Layer"
  • Step 3: Type "Ground" in an empty layer slot
  • Step 4: Select your Player object
  • Step 5: In the Inspector, set "What Is Ground" to "Ground" layer
  • Step 6: Select your Platform object
  • Step 7: Change its Layer from "Default" to "Ground"
⚠️ Critical Step: Don't forget Step 7! If you create the Ground layer but don't assign it to your platform, the raycast will never detect it. This is a very common mistake!

Why Layers Matter

🛡️ Layer Benefits

  • Performance: Raycast only checks relevant objects, not everything
  • Precision: Detect exactly what you want, ignore what you don't
  • Flexibility: Different raycasts can detect different layers
  • Organization: Clear categorization of game objects

10. Testing Ground Detection 🧪

Concept:

Once everything is set up, you can test the ground detection by making the isGrounded variable temporarily visible in the Inspector. This lets you see in real-time whether the raycast is detecting the ground.

Temporary Visibility for Testing

Making isGrounded Visible
// For testing only:
[SerializeField] private bool isGrounded;

// After testing, hide it again:
private bool isGrounded;

🎮 What to Look For

  • Standing Still: isGrounded should show "True"
  • Jumping: isGrounded should switch to "False" immediately
  • In Air: isGrounded stays "False" while airborne
  • Landing: isGrounded switches back to "True" upon landing
🔧 Troubleshooting: If isGrounded is always False, check: (1) Platform has Ground layer assigned, (2) groundCheckDistance is long enough, (3) whatIsGround is set to Ground layer, (4) HandleCollision() is being called in Update().

Adjusting Distance Based on Testing

Finding the Right Value
// Too short - won't detect ground:
groundCheckDistance = 0.1;

// Good range for most characters:
groundCheckDistance = 0.4 to 1.4;

// The raycast must reach from character center to slightly into the ground

11. Implementing Jump Restrictions 🚫

Concept:

Now that we can detect the ground, we use that information to control when jumping is allowed. A simple if statement checks the isGrounded variable before applying jump force.

Updated Jump Method

Jump with Ground Check
private void Jump()
{
    if (isGrounded)
    {
        rb.velocity = new Vector2(rb.velocity.x, jumpForce);
    }
}

✅ How It Works

  • Ground Check First: Before jumping, check if isGrounded is true
  • Permission Gate: If true, allow jump; if false, do nothing
  • Button Mashing Protected: Player can spam jump button—it won't work in air
  • Clean Physics: Jump only happens when physically possible

Testing the Restriction

🎮 Expected Behavior

  • Standing on ground → Press jump → Character jumps ✅
  • In the air → Press jump → Nothing happens ✅
  • Mashing jump button → Only jumps once per landing ✅
  • No more flying by holding jump button ✅
🎯 Success! Your character now has realistic jump physics that prevents air-jumping. This is a fundamental mechanic in professional platformers!

12. Fine-Tuning Physics Parameters ⚙️

Concept:

The Rigidbody2D component has several physics parameters that dramatically affect how your character feels to control. "Game feel" is all about tweaking these values until movement feels responsive and satisfying.

Gravity Scale

Adjusting Gravity
// In Rigidbody2D component:
Gravity Scale: 3.5

// Higher values = falls faster (feels heavier)
// Lower values = floaty feeling

🌍 Gravity Scale Effects

  • 1.0: Default Unity gravity—usually too floaty
  • 2.0-3.0: Lighter, more arcade-like feel
  • 3.5-4.5: Snappy, responsive feel (recommended for platformers)
  • 5.0+: Very fast falling—can feel too heavy

Linear Drag (Air Resistance)

Adding Air Resistance
// In Rigidbody2D component:
Linear Drag: 0.5

// Simulates air resistance—character slows down naturally

💨 Linear Drag Effects

  • 0: No air resistance—character slides forever
  • 0.1-0.5: Subtle resistance—feels natural
  • 1.0+: Strong resistance—movement feels sluggish
  • Recommendation: 0.5 for a slight, realistic feel

Jump Force

Adjusting Jump Height
// In your script:
[SerializeField] private float jumpForce = 12f;

// Test different values to find what feels right

🚀 Jump Force Guidelines

  • 5-8: Small hops—good for quick, nimble characters
  • 10-15: Medium jumps—versatile range for most platformers
  • 12: Recommended starting point for testing
  • 20+: Very high jumps—use for super jump abilities

The Recommended Setup

Optimal Starting Values
// Rigidbody2D settings:
Gravity Scale: 3.5
Linear Drag: 0.5

// Script variables:
jumpForce = 12f;
groundCheckDistance = 0.4f to 1.4f; (depends on character size)
🎨 Game Feel is Subjective: These values are starting recommendations. The best approach is to playtest extensively and adjust based on how YOUR game feels. Try different combinations and see what matches your vision!

13. Understanding Raycast vs Gizmos 🔄

Concept:

This is a common source of confusion for beginners: Gizmos and Raycasts are completely separate systems. Gizmos are for visualization only. Raycasts do the actual physics detection. You need both, with matching parameters, for a complete debugging setup.

Two Separate Systems

Gizmos vs Raycast
// GIZMOS - Only draws a visible line (no detection):
private void OnDrawGizmos()
{
    Gizmos.DrawLine(
        transform.position,
        transform.position + new Vector3(0, -groundCheckDistance, 0)
    );
}

// RAYCAST - Does actual physics detection (invisible):
private void HandleCollision()
{
    isGrounded = Physics2D.Raycast(
        transform.position,
        Vector2.down,
        groundCheckDistance,
        whatIsGround
    );
}

🎯 Key Differences

  • Gizmos: Visual only • Visible in Scene view • No physics calculation
  • Raycast: Physics detection • Invisible to everyone • Returns true/false
  • Both Use Same Values: Same origin, same distance for accurate debugging
  • Can Work Independently: Raycast works without gizmos, but gizmos help debugging
🔍 Debugging Workflow: Draw gizmos to SEE where your raycast is shooting. If the gizmo line looks correct but detection isn't working, the problem is with your raycast parameters or layer setup, not the gizmo.

Why Use Both?

🛠️ The Perfect Debugging Combo

  • Visual Confirmation: See exactly where and how far you're checking
  • Easy Adjustment: Change groundCheckDistance and see instant visual feedback
  • No Guessing: Know immediately if your distance is too short or too long
  • Professional Practice: Industry standard for debugging invisible systems

14. Cleaning Up Inspector Variables 🧹

Concept:

Once you've verified that systems work, hide debugging variables from the Inspector to keep it clean. Only expose variables that need adjustment during development or design iteration.

What to Hide vs Show

Clean Inspector Practice
// HIDE - Internal state (working correctly):
private bool facingRight;
private float xInput;
private bool isGrounded;

// SHOW - Designer needs to adjust these:
[SerializeField] private float moveSpeed;
[SerializeField] private float jumpForce;
[SerializeField] private float groundCheckDistance;
[SerializeField] private LayerMask whatIsGround;

✨ Inspector Hygiene

  • Hide Working Systems: If isGrounded works, make it private
  • Show Tunable Values: Speed, jump height, distances need adjustment
  • Show Required References: Layer masks, prefab references must be assigned
  • Group with Headers: Related variables under clear section names
🎯 Pro Practice: A clean Inspector means faster iteration. Designers and artists can find and adjust the values they need without digging through debug variables.

Exercise 3: Physics Feel Testing

Task: Compare different physics parameter combinations

  • Test: Gravity 1.0, Jump 12 vs Gravity 3.5, Jump 12
  • Test: Linear Drag 0 vs Linear Drag 2.0
  • Create 3 different "feels": Floaty, Snappy, Heavy
  • Note which combination feels best for your game idea

Exercise 4: Debug Multiple Platforms

Task: Test ground detection with multiple platforms

  • Create 3-5 platforms at different heights in your scene
  • Ensure ALL platforms have the "Ground" layer assigned
  • Jump between platforms and verify isGrounded changes correctly
  • Try jumping while falling through platforms—what happens?

Challenge Exercise: Coyote Time

Task: Allow jumping briefly after leaving the ground

  • Research "coyote time" in game design
  • Create a float variable: coyoteTimeCounter
  • When leaving ground, start counting down (use Time.deltaTime)
  • Allow jumping if counter > 0, even if not grounded
  • This makes platforming feel more forgiving!
