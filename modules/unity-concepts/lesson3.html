1. Understanding If Statements 🤔

Concept:

If statements allow your program to make decisions based on conditions. If a condition is true, the code inside the if statement executes.

Basic Syntax:

If Statement Structure
if (condition)
{
    // Code to execute if condition is true
}

💡 How It Works:

  1. Write if followed by parentheses
  2. Inside parentheses, write a condition that returns true or false
  3. Open curly brackets {}
  4. Write the code that should run if the condition is true

Simple Example:

Example: Basic If Statement
if (Input.GetKey(KeyCode.Q))
{
    Debug.Log("Someone holding Q key");
}
When Q is pressed:
Someone holding Q key
💡 Tip: The condition inside the parentheses must evaluate to true or false. If it's true, the code in curly brackets runs. If it's false, the code is skipped.

2. Unity's Old Input System ⌨️

Concept:

Unity's old Input System provides methods to detect keyboard and mouse input. There are three main methods for keyboard input:

Method When It Returns True Best For
Input.GetKey() While user holds down the key Continuous actions (moving)
Input.GetKeyDown() When user first presses the key Immediate actions (jumping)
Input.GetKeyUp() When user releases the key Release actions (rare)

📝 Input.GetKey() - Continuous Press:

Example: Detecting Held Key
void Update()
{
    if (Input.GetKey(KeyCode.Q))
    {
        Debug.Log("Someone holding Q key");
    }
}
While Q is held:
Someone holding Q key
Someone holding Q key
Someone holding Q key
(Message repeats every frame)

💡 When to Use:

GetKey() returns true every frame while the key is held. Perfect for continuous movement.

📝 Input.GetKeyDown() - Single Press:

Example: Detecting Key Press Start
void Update()
{
    if (Input.GetKeyDown(KeyCode.Q))
    {
        Debug.Log("Someone pressed Q key");
    }
}
When Q is first pressed:
Someone pressed Q key
(Only once, even if held)

💡 When to Use:

GetKeyDown() returns true only on the frame when the key is first pressed. Perfect for actions like jumping, shooting, or interacting.

📝 Input.GetKeyUp() - Key Release:

Example: Detecting Key Release
void Update()
{
    if (Input.GetKeyUp(KeyCode.Q))
    {
        Debug.Log("Someone released Q key");
    }
}
When Q is released:
Someone released Q key

💡 When to Use:

GetKeyUp() returns true only when the user releases the key. Rarely used, but useful for charge-up mechanics or bow-and-arrow systems.

3. Implementing Jump Mechanics 🦘

Concept:

To make a player jump, we need to detect when the jump button is pressed and then modify the player's vertical velocity (Y-axis).

Step 1: Create Jump Force Variable

Serialize Field for Jump Force
[SerializeField] private float jumpForce = 8f;
private Rigidbody2D rb;

💡 Why SerializeField?

Using [SerializeField] allows you to adjust the jump force in Unity's Inspector while keeping the variable private. This gives you flexibility during testing without exposing the variable publicly.

Step 2: Detect Jump Input

Detect Spacebar Press
void Update()
{
    if (Input.GetKeyDown(KeyCode.Space))
    {
        // Make the player jump
    }
}
💡 Tip: We use GetKeyDown() instead of GetKey() because we want the jump to happen once per button press, not continuously while the button is held.

Step 3: Modify Velocity to Jump

Complete Jump Implementation
void Update()
{
    if (Input.GetKeyDown(KeyCode.Space))
    {
        rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
    }
}

💡 How It Works:

  1. rb.linearVelocity.x - Keep the current horizontal velocity (don't interrupt movement)
  2. jumpForce - Set the vertical velocity to the jump force (makes player go up)
  3. new Vector2() - Create a new velocity vector with both X and Y components

Tuning Jump Feel:

Adjusting Jump Parameters
// Higher jumps, faster fall
[SerializeField] private float jumpForce = 25f;
// Gravity scale in Rigidbody2D: 5

// Floaty jumps, slower fall
[SerializeField] private float jumpForce = 8f;
// Gravity scale in Rigidbody2D: 1.5

💡 Jump Tuning Tips:

  • Jump Force: Higher values = higher jumps
  • Gravity Scale: Higher values = faster fall, snappier feel
  • Low Gravity + Low Jump Force: Floaty, moon-like feel
  • High Gravity + High Jump Force: Snappy, responsive feel
⚠️ Known Issue: Multiple jumps (air jumping) is possible with this implementation. We'll fix this later with ground detection. For now, this behavior is expected.

4. Code Organization with Functions 🧹

Concept:

Clean code is readable and maintainable. One key principle: "If you can extract something into a method, you should do it."

Before Refactoring:

Messy Update Method
void Update()
{
    // Movement input
    float xInput = Input.GetAxis("Horizontal");
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);

    // Jump input
    if (Input.GetKeyDown(KeyCode.Space))
    {
        rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
    }
}

❌ Problems:

  • Update method has too many responsibilities
  • Difficult to read and understand at a glance
  • Hard to debug and maintain

Step 1: Extract Jump into Function

Create Jump Method
private void Jump()
{
    rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
}

void Update()
{
    // Movement input
    float xInput = Input.GetAxis("Horizontal");
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);

    // Jump input
    if (Input.GetKeyDown(KeyCode.Space))
        Jump();
}

✅ Benefits:

  • Update is now easier to read: "If space is pressed, Jump()"
  • Jump logic is separated and reusable
  • If you need to see jump details, you can look at the Jump() method

Step 2: Extract Movement into Function

Create HandleMovement Method
private void HandleMovement()
{
    float xInput = Input.GetAxis("Horizontal");
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);
}

void Update()
{
    HandleMovement();

    if (Input.GetKeyDown(KeyCode.Space))
        Jump();
}

Step 3: Extract All Input into Function

Final Clean Structure
private void HandleInput()
{
    // Get movement input
    xInput = Input.GetAxis("Horizontal");

    // Check for jump input
    if (Input.GetKeyDown(KeyCode.Space))
        Jump();
}

private void HandleMovement()
{
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);
}

private void Jump()
{
    rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
}

void Update()
{
    HandleInput();
    HandleMovement();
}

✅ Final Structure Benefits:

  • Update(): Clean overview of what happens each frame
  • HandleInput(): All input detection in one place
  • HandleMovement(): Movement logic separated
  • Jump(): Jump logic separated and reusable
📖 Clean Code Principle: You read code more than you write it. Make it easy to read!

5. Else If and Else Statements 🔀

Concept:

Sometimes you need alternative conditions or a default action. That's where else if and else come in.

Else If - Alternative Conditions:

Multiple Jump Keys Example
if (Input.GetKeyDown(KeyCode.Space))
{
    Jump();
}
else if (Input.GetKeyDown(KeyCode.UpArrow))
{
    Jump();
}

💡 How It Works:

  1. First checks if Space was pressed
  2. If not, checks if UpArrow was pressed
  3. Only one condition will execute

Else - Default Action:

Else Statement Example
if (Input.GetKeyDown(KeyCode.Space))
{
    Jump();
}
else
{
    Debug.Log("No jump button pressed");
}

💡 Else Statement:

  • No condition needed - it's the "default" case
  • Runs only if all previous conditions are false
  • Always comes at the end
💡 Tip: You don't always need else statements. Use them when you need a default action or when all cases must be handled.

6. Single-Line If Statements ✨

Concept:

When an if statement has only one line of code, you can skip the curly brackets. This is a stylistic choice.

With Curly Brackets:

Standard Format
if (Input.GetKeyDown(KeyCode.Space))
{
    Jump();
}

Without Curly Brackets:

Single-Line Format
if (Input.GetKeyDown(KeyCode.Space))
    Jump();

⚠️ Important Rule:

Only works for ONE line of code. If you add a second line, only the first line is part of the if statement!

❌ Common Mistake:

This Doesn't Work As Expected!
if (Input.GetKeyDown(KeyCode.Space))
    Jump();
    Debug.Log("Jumped!"); // This ALWAYS runs!
Result:
The Debug.Log runs every frame, not just when jumping!

✅ Correct Version:

Use Curly Brackets for Multiple Lines
if (Input.GetKeyDown(KeyCode.Space))
{
    Jump();
    Debug.Log("Jumped!");
}

💡 Style Preference:

  • No brackets: Saves space, clean for simple conditions
  • With brackets: More consistent, prevents mistakes
  • Recommendation: Use whatever style you prefer, but be consistent!

7. Extract Method Tool (VS Code) 🛠️

Concept:

Visual Studio Code has a powerful refactoring tool called "Extract Method" that automatically creates functions from selected code.

How to Use Extract Method:

Steps:

  1. Select the code you want to extract into a method
  2. Right-click and choose "Quick Actions and Refactorings"
  3. Or press Alt + Enter (Windows) or Ctrl + .
  4. Select "Extract Method"
  5. Name your new method

Example - Before:

Original Code
void Update()
{
    // Select these lines ↓
    float xInput = Input.GetAxis("Horizontal");
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);

    if (Input.GetKeyDown(KeyCode.Space))
        Jump();
}

Example - After:

After Extract Method → "HandleMovement"
void Update()
{
    HandleMovement();

    if (Input.GetKeyDown(KeyCode.Space))
        Jump();
}

private void HandleMovement()
{
    float xInput = Input.GetAxis("Horizontal");
    rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);
}

💡 Benefits of Extract Method:

  • Automatically creates the method signature
  • Handles parameters and return values
  • Fast and reduces errors
  • Works in Visual Studio and VS Code
⚠️ Mac Users (VS Code): If Extract Method doesn't work, try installing the "Visual Studio Editor" package in Unity's Package Manager or the "OmniSharp" extension in VS Code. If it still doesn't work, you can always create methods manually!

8. Complete Player Controller Example 🎯

Full Refactored Player Controller
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [SerializeField] private float moveSpeed = 5f;
    [SerializeField] private float jumpForce = 8f;

    private Rigidbody2D rb;
    private float xInput;

    void Start()
    {
        rb = GetComponent<Rigidbody2D>();
    }

    void Update()
    {
        HandleInput();
        HandleMovement();
    }

    private void HandleInput()
    {
        xInput = Input.GetAxis("Horizontal");

        if (Input.GetKeyDown(KeyCode.Space))
            Jump();
    }

    private void HandleMovement()
    {
        rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);
    }

    private void Jump()
    {
        rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
    }
}

✅ Clean Code Features:

  • ✅ Update(): Easy to read overview
  • ✅ HandleInput(): All input in one place
  • ✅ HandleMovement(): Movement logic separated
  • ✅ Jump(): Reusable jump method
  • ✅ Serialized Fields: Tunable in Inspector
