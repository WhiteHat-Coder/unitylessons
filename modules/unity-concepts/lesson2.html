1. C# Variables - Your Data Storage 📦

What is a Variable?

Variable = a labeled box that stores data

Real-life analogy: Think of your phone's contacts. Each contact has:

  • A name (label) → "Mom"
  • A phone number (data) → "555-1234"
  • A type (what kind of data) → Phone Number

Variables work the same way in coding!

📝 Variable Syntax:

Example: Declaring a Variable
public string playerName = "Bob";

🧠 Breaking Down the Syntax:

  • public → Access modifier (who can see this variable)
  • string → Type (what kind of data: text in this case)
  • playerName → Name (the label for your box)
  • = "Bob" → Value (the actual data stored)

📋 The 4 Core Data Types:

1️⃣ string → Text/Words

Example: String Variables
public string playerName = "Bob";
public string weaponType = "Sword";
public string enemyName = "Goblin";

void Start()
{
    Debug.Log(playerName + " equipped a " + weaponType);
}
Output:
Bob equipped a Sword

2️⃣ int → Whole Numbers (no decimals)

Example: Integer Variables
public int playerAge = 25;
public int currentHP = 100;
public int score = 0;

void Start()
{
    Debug.Log("Health: " + currentHP);
    currentHP = currentHP - 25; // Take damage
    Debug.Log("Health after damage: " + currentHP);
}
Output:
Health: 100
Health after damage: 75

3️⃣ float → Decimal Numbers

Example: Float Variables
public float moveSpeed = 2.5f; // 'f' at end means float!
public float jumpHeight = 3.2f;
public float temperature = 98.6f;

void Start()
{
    Debug.Log("Move speed: " + moveSpeed);
}
Output:
Move speed: 2.5
⚠️ Important: Always add f at the end of float numbers!
✅ Correct: 2.5f
❌ Wrong: 2.5 (Unity thinks it's a double, not float)

4️⃣ bool → True/False

Example: Boolean Variables
public bool isGameOver = false;
public bool hasKey = true;
public bool isGrounded = true;

void Start()
{
    if (hasKey)
    {
        Debug.Log("Player can open the door!");
    }
}
Output:
Player can open the door!

❌ Common Errors:

Example: Type Mismatch Errors
// ❌ ERROR: Can't store text in an int
public int age = "25"; // Wrong!

// ❌ ERROR: Can't store number in a string
public string name = 25; // Wrong!

// ❌ ERROR: Missing 'f' for float
public float speed = 2.5; // Wrong!

// ✅ CORRECT:
public int age = 25;
public string name = "Bob";
public float speed = 2.5f;

💡 Quick Reference Table:

Type Stores Example
string Text "Hello"
int Whole numbers 42
float Decimals 3.14f
bool True/False true

2. Unity Execution Order (Lifecycle) 🔄

What is the Unity Lifecycle?

Unity calls specific functions automatically at different times. You just fill in what happens!

Think of it like a daily routine:

  • 🌅 Wake up (Awake)
  • 🍳 Eat breakfast (Start)
  • 📝 Work all day, every minute (Update)
  • ⏰ Regular breaks every hour (FixedUpdate)

📝 The 4 Important Functions:

Example: Lifecycle Methods
using UnityEngine;

public class LifecycleExample : MonoBehaviour
{
    void Awake()
    {
        Debug.Log("1. Awake - Called FIRST, before anything else");
    }

    void Start()
    {
        Debug.Log("2. Start - Called after Awake, before first Update");
    }

    void Update()
    {
        Debug.Log("3. Update - Called EVERY FRAME (60+ times per second!)");
    }

    void FixedUpdate()
    {
        Debug.Log("4. FixedUpdate - Called at FIXED intervals (50 times/sec)");
    }
}
Output (first 2 frames):
1. Awake - Called FIRST, before anything else
2. Start - Called after Awake, before first Update
3. Update - Called EVERY FRAME
4. FixedUpdate - Called at FIXED intervals
3. Update - Called EVERY FRAME
3. Update - Called EVERY FRAME
...

🧠 When to Use Each Method:

Method When Called Use For
Awake() Once at start (first) Getting components, initial setup
Start() Once before first frame Initialization after Awake
Update() Every frame Input, movement, game logic
FixedUpdate() Fixed time intervals Physics (Rigidbody movement)

⚠️ The FPS Problem:

Problem: Update() runs every frame. Different computers have different frame rates!

  • 💻 Gaming PC: 144 FPS → Update runs 144 times/second
  • 📱 Old phone: 30 FPS → Update runs 30 times/second

This means: Same movement code runs at different speeds on different devices!

Example: Why Update Can Be Problematic
void Update()
{
    // ❌ BAD: Moves faster on high FPS computers!
    transform.position += Vector3.forward * 5;
}

void Update()
{
    // ✅ GOOD: Same speed on all computers!
    transform.position += Vector3.forward * 5 * Time.deltaTime;
}
💡 Golden Rule:
• Use Update() for input and general logic
• Use FixedUpdate() for physics (Rigidbody)
• Always multiply movement by Time.deltaTime in Update!

🔍 What is MonoBehaviour?

MonoBehaviour is the base class all Unity scripts inherit from.

Think of it like this: MonoBehaviour gives your script superpowers!

  • ✅ Lets Unity call Awake(), Start(), Update()
  • ✅ Allows script to be attached to GameObjects
  • ✅ Gives access to transform, gameObject, etc.
Example: MonoBehaviour is Required
// ✅ CORRECT: Inherits from MonoBehaviour
public class Player : MonoBehaviour
{
    void Start() { } // Unity can call this!
}

// ❌ WRONG: Can't attach to GameObject without MonoBehaviour
public class Player
{
    void Start() { } // Unity won't call this!
}

3. Functions (Methods) - Code Reusability 🔧

What is a Function?

Function = a reusable block of code that performs a specific task

Real-life analogy: Making a sandwich

  • 📖 Recipe (function definition): How to make it
  • 🍞 Making sandwich (calling function): Actually doing it

You write the recipe once, but can make sandwiches many times!

📝 Basic Function:

Example: Simple Function
public class Player : MonoBehaviour
{
    // Define the function
    void GetPlayerInfo()
    {
        Debug.Log("Player Name: Bob");
        Debug.Log("Player Level: 5");
    }

    void Start()
    {
        // Call the function
        GetPlayerInfo();
        GetPlayerInfo(); // Can call multiple times!
    }
}
Output:
Player Name: Bob
Player Level: 5
Player Name: Bob
Player Level: 5

📥 Functions with Parameters:

Example: Function with Input
public class Player : MonoBehaviour
{
    public int currentHP = 100;

    // Function takes 'damage' as input
    void TakeDamage(int damage)
    {
        currentHP = currentHP - damage;
        Debug.Log("Took " + damage + " damage! HP: " + currentHP);
    }

    void Start()
    {
        TakeDamage(25); // Take 25 damage
        TakeDamage(10); // Take 10 more damage
    }
}
Output:
Took 25 damage! HP: 75
Took 10 damage! HP: 65

💡 DRY Principle (Don't Repeat Yourself):

Without Functions: Copy-paste same code everywhere

currentHP = currentHP - 25;
Debug.Log("HP: " + currentHP);

currentHP = currentHP - 10;
Debug.Log("HP: " + currentHP);

currentHP = currentHP - 30;
Debug.Log("HP: " + currentHP);

With Functions: Clean and reusable

TakeDamage(25);
TakeDamage(10);
TakeDamage(30);

4. Player Input & Movement 🎮

Understanding Movement:

In 2D games, movement happens on two axes:

  • X-axis → Left (-) / Right (+)
  • Y-axis → Down (-) / Up (+)

📝 Reading Input:

Example: Detecting Input
void Update()
{
    float xInput = Input.GetAxisRaw("Horizontal");
    Debug.Log("Input: " + xInput);
}

🎹 Input Values:

  • -1 → Left arrow or A key pressed
  • 0 → No input (nothing pressed)
  • 1 → Right arrow or D key pressed

🚀 Building Complete Player Movement:

Step 1: Setup (In Unity Editor)

  1. Create a 2D Sprite (Player)
  2. Add Rigidbody2D component
  3. Set Gravity Scale to 1
  4. Add Box Collider 2D
  5. Create new C# script called "PlayerMovement"
  6. Attach script to Player GameObject

Step 2: Write the Movement Script

Complete Movement Script
using UnityEngine;

public class PlayerMovement : MonoBehaviour
{
    // Reference to Rigidbody component
    private Rigidbody2D rb;
    
    // Store input value
    private float xInput;
    
    // Movement speed (can adjust in Inspector!)
    [SerializeField] private float moveSpeed = 3.5f;

    void Awake()
    {
        // Get the Rigidbody2D component
        rb = GetComponent<Rigidbody2D>();
    }

    void Update()
    {
        // Read input every frame
        xInput = Input.GetAxisRaw("Horizontal");
    }

    void FixedUpdate()
    {
        // Apply movement to Rigidbody
        rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);
    }
}

🧠 Breaking Down the Code:

1. Variables:

private Rigidbody2D rb;

Stores reference to the Rigidbody2D component

private float xInput;

Stores the input value (-1, 0, or 1)

[SerializeField] private float moveSpeed = 3.5f;

Movement speed - shown in Inspector because of [SerializeField]

2. Getting the Component:

rb = GetComponent<Rigidbody2D>();

Finds and stores the Rigidbody2D component attached to this GameObject

3. Reading Input (Update):

xInput = Input.GetAxisRaw("Horizontal");

Checks if A/D or Arrow keys are pressed

Why Update()? Input should be checked every frame to feel responsive

4. Applying Movement (FixedUpdate):

rb.linearVelocity = new Vector2(xInput * moveSpeed, rb.linearVelocity.y);

Breaking it down:

  • new Vector2(x, y) → Creates a 2D movement vector
  • xInput * moveSpeed → Horizontal movement (left/right)
  • rb.linearVelocity.y → Keeps vertical movement unchanged (for jumping/falling)

Why FixedUpdate()? Because we're using Rigidbody (physics)

⚡ Velocity vs Position:

Velocity (Speed) Position
Smooth, physics-based Instant teleportation
Works with collisions Can pass through walls
Affected by gravity Not affected by physics
✅ Use for player Use for UI, camera

🎯 Test Your Movement:

  1. Press Play ▶️ in Unity
  2. Press A or ← → Character moves left
  3. Press D or → → Character moves right
  4. Release keys → Character stops
  5. Adjust moveSpeed in Inspector to change speed

5. Clean Code Practices 🧹

🔐 Access Modifiers:

Public vs Private:

public private
Visible everywhere Only visible inside this script
Other scripts can access Hidden from other scripts
Shows in Inspector Hidden in Inspector
Example: Access Modifiers
public class Player : MonoBehaviour
{
    public int maxHealth = 100; // ✅ Visible everywhere
    private int currentHealth = 100; // 🔒 Hidden (internal use only)

    public void TakeDamage(int damage) // ✅ Other scripts can call
    {
        ReduceHealth(damage); // Call private helper
    }

    private void ReduceHealth(int amount) // 🔒 Internal helper
    {
        currentHealth -= amount;
    }
}

🎯 Best Practice: SerializeField

Problem: You want private variables (clean code) BUT also want to edit them in Inspector

Solution: Use [SerializeField]

Example: SerializeField
public class Player : MonoBehaviour
{
    // ✅ BEST: Private but visible in Inspector
    [SerializeField] private float moveSpeed = 3.5f;
    [SerializeField] private int maxHealth = 100;

    // ❌ Avoid this (breaks encapsulation)
    public float moveSpeed = 3.5f;
}

🔧 Component References:

Always get components in Awake() for better performance

Example: Getting Components
private Rigidbody2D rb;
private SpriteRenderer spriteRenderer;
private Animator animator;

void Awake()
{
    // Get all components once at start
    rb = GetComponent<Rigidbody2D>();
    spriteRenderer = GetComponent<SpriteRenderer>();
    animator = GetComponent<Animator>();
}
💡 Why Awake() for GetComponent?
• Only runs once (efficient)
• Guarantees component is found before Start()
• Better than calling GetComponent every frame

Question 1: What is a variable?

Answer: A labeled container/box that stores data

Question 2: What's the difference between int and float?

Answer:

  • int → Whole numbers only (25, 100, -5)

Answer: Every frame (typically 60+ times per second)

Answer:
